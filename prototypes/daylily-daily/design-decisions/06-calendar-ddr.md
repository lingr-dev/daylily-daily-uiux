# Design Decisions Record (DDR) - 萱草日签 日历视图 (Calendar View)

## 1. 核心设计哲学：化繁为简，放大感知

**[决策背景]**
日历/热力图是健康追踪类产品的标配，但传统的 "GitHub Style" 密集热力图对于中老年用户（视力衰退、触控精度下降）非常不友好。需要一种既能展示全貌，又适老化的方案。

**[设计共识]**
- **疏网格设计：** 摒弃全年密集网格，采用单月 7 列宽幅网格（周日～周六），保证每一个日期格子具有至少 `w-12 h-12` 及以上的有效点击热区，内部使用 `18px` 以上的超大字体。
- **色彩疗愈与警示：** 
  - 通过大面积的“森林绿”色块填铺成就感。
  - 通过“暖黄”标识局部异常（如晚点补服），保留柔性提醒意味。
  - 尽量避免强烈的报错红，极端漏服使用红色轮廓或微小红点，不剥夺用户的核心关注区间，降低医疗焦虑。

## 2. 界面结构机制

### 2.1 月度高光总结 (Monthly Hero Area)
- 提供明确的“本月已坚持 XX 天”统计正反馈。
- 排布在屏幕视觉第一落点，强化持续使用的动力。

### 2.2 7 列日历热力网格 (The Grid)
- **视觉重心：** 所有发生过事件的日期，均按圆角色块处理。未来的日子采用无色彩边界以平息注意力。
- **独立选中态 (Decoupled Selection State) [NEW]：** 
  - 弃用传统的“内部覆盖式色值/边框”。为了保证日期原本的健康状态颜色（绿/黄）绝对不被遮掩或篡改，我们提炼了独立的 `.day-selected` 样式交互类。
  - **实现方案：** 巧妙利用 CSS 伪元素 (`::after`) 在目标元素向外扩出 `4px` 的物理边缘外，绘制一圈高对比度的深色光环（Ring）。这样“状态表达”与“焦点定位”就成了互不干扰的双层叠态，且附带点击回弹的弹簧微动效 (`scale-105`)，大大增强了触觉反馈。

### 2.3 底部详情流展示 (Detail Timeline)
- 点击任意网格后，下半屏无缝、且带有淡入动画（Fade-in）地展开该日详情流。
- 完全复用首页的 Timeline 组件规范，保持视觉一致性。
