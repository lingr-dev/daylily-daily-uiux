# Design Decisions Record (DDR) - 萱草日签 UI/UX 核心共识

## 1. 核心设计哲学：我优先 (Me-First)
**[决策背景]**
在家庭照护场景下，照护者往往容易被他人的事务淹没，产生照护焦虑。

**[设计共识]**
- **个人中心锚点：** 首页视觉与交互的绝对中心永远是“我”的用药任务。
- **情感化反馈：** 首页大按钮不仅是打卡工具，更是“自我健康成就感”的载体。
- **异常驱动：** 家人的漏服情况仅作为“异常流”在次级区域提醒，不篡夺首页大按钮的控制权。

## 2. 视觉语言规范 (Visual Identity)

### 2.1 适老化动态排版体验 (Dynamic Accessibility Typography) [REVISED]
- **放弃极端静态大字号：** 坚决废弃硬编码巨大的字体（如 30px、44px）。因为微信自带“关怀模式”字体缩放，极巨化底盘会二次倍增进而导致界面错乱截断。
- **动态基准缩放机制：** 设计以 `16px - 18px` (text-base / text-lg) 作为正文和重要提示基础，把放大决策权交还给微信操作系统引擎。
- **弹性布局容器：** 全面弃用写死的容器高度（`h-xx`），转而广泛使用 `min-h-[xxx]` 结合内边距，确保文字由于系统机制猛烈变大换行时，容器能顺畅自然地垂直拉伸。
- **维持宽裕触控区：** 字号可以克制，但胖手指“触控热区”必须保底，核心按钮、交互卡片和操作图标其触控区至少保持在 `44px` 到 `56px`。

### 2.2 全局调性：温暖居家 (Warm & Domestic)
- **全局背景色：** 采用 **#F9F7F3 (宣纸色/微暖沙色)**，摒弃生硬的纯白，营造出更具亲和力的“居家感”，让高饱和度的品牌色更好地融入环境。
- **呼吸感：** 使用带有柔和光晕的 UI 元素，通过 `box-shadow` 和透明度动画实现“呼吸感”，避免机械的工业感。

### 2.2 核心色彩体系
- **品牌主色：** 暖橙色 (#FF9500)，象征阳光、活力与关怀。
- **状态辅助色：** 森林绿 (#34C759) 代表达标成功；警示红 (#FF3B30) 代表异常漏服。

## 3. 首页交互结构 (Home Screen Architecture)

### 3.1 核心组件：动态英雄按钮 (The Hero Button)
- **尺寸与排版：** 核心交互区维持高占比，内部采用特大字号（24px 主标题），兼顾视觉冲击力与适老化。
- **外围进度环 (Activity Ring) [NEW]：**
  - 在按钮外部包裹基于 SVG 的细腻环形进度条（类似 Apple Watch 表盘），伴随打卡动作动态填充，清晰展示今日照护目标的达成进度，增强“成就感”。
- **场景化状态机制 (Time-gated State Machine) [NEW]：**
  摒弃无限随时打卡逻辑，严格基于时间窗进行交互控制，并辅以柔性安抚文案：
  1. **未到时间 (Waiting)：** 距计划 > 1小时。置灰，不可点击。文案：“安心休息”。
  2. **准时打卡 (Ready)：** 计划前 1 小时内。暖橙色，可点击。文案：“我吃过了”。
  3. **临近超时 (Warning)：** 计划后 1 小时内。暖黄色，可点击。文案：“我补吃了”并辅以宽慰语（稍晚了一点哦）。
  4. **已经错过 (Missed)：** 计划后 > 1小时。首页按钮刷新至下一计划，本次漏服任务掉落至时间轴以示警。
  5. **今日达成 (All Done)：** 当日任务清空。森林绿，不可点击。文案：“今日已达标”。
  6. **个人空窗态 (Idle - Pure Caregiver) [V1.4, REVISED]：** 当全局个人计划数为 0，但家人计划数 > 0 时。琥珀色 (Amber-400)，**不可点击**。文案：“记录我的健康”，副标题：“先照顾好自己”。为自己添加用药计划：**「我的」Tab → 日常用药计划（「我的用药」）**，与 `specs/01-home-screen-contract.md` §1 统一入口一致。
  7. **今日无排程 Idle [SPEC]：** 当个人计划数 > 0 但 **当日内无任何排程** 时，Hero 进入 Idle 表现（非 All Done），**不可点击**；进度环 idle 视觉；副文案建议「今日暂无用药安排」等；管理计划同 **「我的」— 日常用药计划**。

- **微动效规范 (Subtle Micro-animations) [V1.4]：** 
  - 核心大按钮的物理实体保持静止不动，转而在其背后的环境光晕 (Hero Glow) 上应用具有生命力的扩张与透明度起伏动效 (`glow-breath`，周期 3s)。**仅 Ready / Warning 等可点击态开启**；Waiting、All Done、**两种 Idle** 关闭光晕呼吸。这种间接的视觉暗示既能保持按钮交互的绝对稳定感，又提供了充足的情绪价值。

### 3.2 导航：灵动岛视图切换 (Identity Toggle)
- **形态：** 导航栏下方的深色半透明胶囊组件。
- **逻辑：** 用于在“我的用药”与“家人照护”两个视角间无感切换。
- **异常感知：** 当家人出现漏服时，胶囊右侧出现警示红点，实现“静默提醒”。

### 3.3 首页三重业务边界展示逻辑 (Three-Tier Home Logic) [V1.4]
基于用户计划数量的不同组合，首页动态切分以下三种主态：
1. **纯净新用户 (Total Plans == 0)：** 隐藏打卡流。全屏展示温润的插画引导页，提供“为自己”与“为家人”双入口。
2. **常规打卡用户 (Personal Plans > 0)：** 默认进入打卡流视图，Hero 按钮根据时间窗自动推演状态。
3. **纯照护者 (Personal Plans == 0, Family Plans > 0)：** Hero 按钮强制锁定为 **个人空窗态 (Idle)**，下方时间轴正常展示家人动态。

### 3.4 健康动态：堆叠卡片流 (Stacked Card Feed) [V1.5 重构]
- **重构背景：** 传统的左侧竖线时间轴（Absolute Line & Dots）在小程序 WebView 中极易出现次像素渲染错位问题，且对多行折行文本（如告警块）的空间包容性极差。
- **全新布局：** 采用宽裕的圆角卡片堆叠（Stacked Cards），移除物理上的连接线，通过纵向视觉流传达时间先后。
- **状态设计与权重展示：**
  1. **异常告警态（如家人漏服）：** 
     - **视觉特征：** 白底卡片 + 警示红细边框 (`border-red-100`) + 右上角微红氛围底色 (`bg-red-50`)。
     - **动效提示：** 头像右上角外挂持续的 `animate-ping` 红色波纹点。
     - **商业化异常嵌件：** 当触发“能量不足导致通知失败”时，卡片内部挂载极简的红色底色告警块。通过极限压缩文案（“能量不足，未发送”+“补充 ➔”）并配合防御性 CSS (`min-w-0`)，确保在 320px 小屏下不换行，原生地引导至能量充值。
  2. **首要聚焦态（如我的当前任务）：** 
     - **视觉特征：** 使用白底卡片配合轻微的品牌色边框，卡片最左侧嵌入一条加粗的主色调锚定线 (`w-1 bg-brand-primary`)，提示“这是当下最高优先级的待办”。
  3. **次要/未来态（如晚间待办）：** 
     - **视觉特征：** 整体降低透明度 (`opacity-70`) 并配合灰阶头像 (`grayscale`)，弱化视觉存在感，绝不干扰当前的主视线流。
- **动作策略：** 首页仅做信息展示，移除所有“一键提醒”等交互按钮，保持信息流纯粹，将操作引导至详情页或线下。

## 4. 微信小程序原生适配边界 (Mini Program Compliance)

### 4.1 顶部导航栏 (NavBar)
- **标准：** 严格遵循 44px (iOS) 原生标题栏高度。
- **安全区：** 右侧预留 87x32px 的胶囊菜单安全区，禁止任何 UI 遮挡。

### 4.2 底部导航栏 (TabBar)
- **结构：** 三段式布局 (日历 - 首页 - 我的)。
- **尺寸：** 1:1 还原原生 TabBar（高度 50px，图标 27px，文字 10px）。
- **适配：** 底部强制留出 `safe-area-inset-bottom` 空间，适配全面屏手势条。

## 6. 版本变更日志 (Changelog)

### V1.5.1 (SPEC 对齐)
- **Hero Idle 不可点：** 纯照护者 Idle 与「今日无排程」Idle 均 **禁止点击**；光晕呼吸仅保留于可打卡态。
- **为自己管计划统一走「我的」：** 除首页 **H-ONBOARD** 冷启动双 CTA 外，为自己创建/管理用药计划 **一律**经 **「我的」→ 日常用药计划（「我的用药」卡片）**；后续多途径再迭代。

### V1.5 (当前版本: 异常流与动态列表重构)
- **堆叠卡片流架构 (Stacked Card Feed)：** 废弃小程序 WebView 中渲染脆弱的连线时间轴，全面拥抱高包容性、稳定渲染的卡片堆叠布局。
- **极限小屏适配 (Pixel-perfect Layout)：** 针对 iPhone SE (320px) 级别的极窄屏幕，引入防御性 `min-w-0` 与 `truncate` 组合，并重写精简了告警文案，根除了 Flex 盒子内联元素挤压换行的顽疾。
- **能量告警闭环 (Energy Exception Strategy)：** 将“提醒能量不足”的异常流作为原生组件嵌入到漏服卡片内，在“成功不打扰，失败必告警”的原则下，顺滑连接商业化补充入口。

### V1.4 (历史版本: 新手引导与边界态闭环)
- **沉浸式新手引导 (Immersive Onboarding)：** 首页新增空窗态，并确立“双入口归流”架构。
- **无缝建联 UX (Contextual Auto-Expansion)：** 彻底改造 `04-plan-create` 页面落地体验。当纯新照护者带着 `?role=family_new` 参数进入时，自动隐藏所有演示数据，平滑展开成员输入框并定位光标，配合“带呼吸动效的悬浮气泡”，在零黑话负担下隐性完成“影子账户”的创建。
- **纯照护者边界覆盖：** 补齐首页“自己无计划，家人有计划”特殊状态下的 Hero Button 闲置态 (Idle) 视觉（琥珀色）；**Hero 跳转新建已废止**（V1.5.1），改走 **「我的」— 日常用药计划**。
- **视觉克制调优：** 为首页大按钮引入极轻量级呼吸动效 (`subtle-breath`)。

### V1.3 (历史版本)
- 基于 SVG 的细粒度进度环与交互动画，及核心大按钮的多态时间机管控。
- 完善灵动岛身份切换与红点静默告警逻辑。
