# Design Decisions Record (DDR) - 萱草日签 UI/UX 核心共识

## 1. 核心设计哲学：我优先 (Me-First)
**[决策背景]**
在家庭照护场景下，照护者往往容易被他人的事务淹没，产生照护焦虑。

**[设计共识]**
- **个人中心锚点：** 首页视觉与交互的绝对中心永远是“我”的用药任务。
- **情感化反馈：** 首页大按钮不仅是打卡工具，更是“自我健康成就感”的载体。
- **异常驱动：** 家人的漏服情况仅作为“异常流”在次级区域提醒，不篡夺首页大按钮的控制权。

## 2. 视觉语言规范 (Visual Identity)

### 2.1 适老化动态排版体验 (Dynamic Accessibility Typography) [REVISED]
- **放弃极端静态大字号：** 坚决废弃硬编码巨大的字体（如 30px、44px）。因为微信自带“关怀模式”字体缩放，极巨化底盘会二次倍增进而导致界面错乱截断。
- **动态基准缩放机制：** 设计以 `16px - 18px` (text-base / text-lg) 作为正文和重要提示基础，把放大决策权交还给微信操作系统引擎。
- **弹性布局容器：** 全面弃用写死的容器高度（`h-xx`），转而广泛使用 `min-h-[xxx]` 结合内边距，确保文字由于系统机制猛烈变大换行时，容器能顺畅自然地垂直拉伸。
- **维持宽裕触控区：** 字号可以克制，但胖手指“触控热区”必须保底，核心按钮、交互卡片和操作图标其触控区至少保持在 `44px` 到 `56px`。

### 2.2 全局调性：温暖居家 (Warm & Domestic)
- **全局背景色：** 采用 **#F9F7F3 (宣纸色/微暖沙色)**，摒弃生硬的纯白，营造出更具亲和力的“居家感”，让高饱和度的品牌色更好地融入环境。
- **呼吸感：** 使用带有柔和光晕的 UI 元素，通过 `box-shadow` 和透明度动画实现“呼吸感”，避免机械的工业感。

### 2.2 核心色彩体系
- **品牌主色：** 暖橙色 (#FF9500)，象征阳光、活力与关怀。
- **状态辅助色：** 森林绿 (#34C759) 代表达标成功；警示红 (#FF3B30) 代表异常漏服。

## 3. 首页交互结构 (Home Screen Architecture)

### 3.1 核心组件：动态英雄按钮 (The Hero Button)
- **尺寸与排版：** 核心交互区维持高占比，内部采用特大字号（24px 主标题），兼顾视觉冲击力与适老化。
- **外围进度环 (Activity Ring) [NEW]：**
  - 在按钮外部包裹基于 SVG 的细腻环形进度条（类似 Apple Watch 表盘），伴随打卡动作动态填充，清晰展示今日照护目标的达成进度，增强“成就感”。
- **场景化状态机制 (Time-gated State Machine) [NEW]：**
  摒弃无限随时打卡逻辑，严格基于时间窗进行交互控制，并辅以柔性安抚文案：
  1. **未到时间 (Waiting)：** 距计划 > 1小时。置灰，不可点击。文案：“安心休息”。
  2. **准时打卡 (Ready)：** 计划前 1 小时内。暖橙色，可点击。文案：“我吃过了”。
  3. **临近超时 (Warning)：** 计划后 1 小时内。暖黄色，可点击。文案：“我补吃了”并辅以宽慰语（稍晚了一点哦）。
  4. **已经错过 (Missed)：** 计划后 > 1小时。首页按钮刷新至下一计划，本次漏服任务掉落至时间轴以示警。
  5. **今日达成 (All Done)：** 当日任务清空。森林绿，不可点击。文案：“今日已达标”。

### 3.2 导航：灵动岛视图切换 (Identity Toggle)
- **形态：** 导航栏下方的深色半透明胶囊组件。
- **逻辑：** 用于在“我的用药”与“家人照护”两个视角间无感切换。
- **异常感知：** 当家人出现漏服时，胶囊右侧出现警示红点，实现“静默提醒”。

### 3.3 任务队列：极简时间轴 (Minimal Timeline)
- **重构：** 摒弃笨重的卡片式布局，采用纤细的时间轴线。
- **权重区分：** 
  - **高亮：** 漏服任务使用红点+扩散波纹动效（Ping Animation）。
  - **中性：** 我的下一个任务使用品牌色圆点。
  - **弱化：** 未来任务采用灰色细体和透明度。
- **动作策略：** 首页仅做信息展示，移除所有“一键提醒”等交互按钮，保持信息流纯粹，将操作引导至详情页或线下。

## 4. 微信小程序原生适配边界 (Mini Program Compliance)

### 4.1 顶部导航栏 (NavBar)
- **标准：** 严格遵循 44px (iOS) 原生标题栏高度。
- **安全区：** 右侧预留 87x32px 的胶囊菜单安全区，禁止任何 UI 遮挡。

### 4.2 底部导航栏 (TabBar)
- **结构：** 三段式布局 (日历 - 首页 - 我的)。
- **尺寸：** 1:1 还原原生 TabBar（高度 50px，图标 27px，文字 10px）。
- **适配：** 底部强制留出 `safe-area-inset-bottom` 空间，适配全面屏手势条。
